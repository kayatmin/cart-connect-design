# CartConnect Mobile brand notes

## Visual Direction

- calm command center for live commerce support
- premium retail restraint, not generic admin software
- mobile operational tool, not marketing site
- compact, exact, and easy to scan during active customer sessions

## Brand System

- Primary identity color: deep ink `#102033`
- Live active color: blue `#2563EB`
- Waiting priority color: amber `#F2A900`
- Clear/success color: green `#167A55`
- Danger is reserved for destructive confirmation and failed actions
- Neutrals should feel cool and clean, with `#F6F7F9` as the base surface

## Typography

- Plus Jakarta Sans for screen titles and strong UI labels
- Source Sans 3 for body and controls
- IBM Plex Mono for compact counts, timestamps, debug labels, and tabular values
- Do not use Inter, Roboto, Poppins, Space Grotesk, or system-ui as the primary look

## UI Characteristics

- 4px status rails for session priority
- compact KPI tiles
- large touch targets
- white and near-white surfaces over a soft neutral app background
- selected state should be obvious without heavy shadows
- bottom navigation may use a filled pill for the active destination
- waiting sessions should be the loudest queue element

## Screen Content Guidance

- Use realistic customer names and retail products
- Include cart totals, chat counts, wait time, and last activity metadata
- Keep auth and network recovery single-purpose
- Avoid generic dashboard filler and lorem ipsum
